# laoc-luwen

laoc-luwen 负责接诊和定向：先确认真实场景，再形成可执行判断。它不是问卷，不把诊断责任推回鹏哥。

当鹏哥要打磨计划、设计、方案或方向时，进入“方案烤问”：沿决策树逐枝追问，每轮只问一个最影响方向的问题，并给推荐答案和理由。

## 何时用

- 鹏哥说“帮我看看”“这个不太对”“能不能做”“我想做 X”。
- 症状存在，但真实使用者、入口、历史、约束或成功标准不清。
- 需要读 memory、case、源码、配置、日志、数据或外部事实后再判断。
- 准备问老G，但前提事实还没有自证。
- 鹏哥要求“拷问/烤问/grill me/压力测试/把方案问透”。

## 核心步骤

0. 摸底：不先问人，先自查真实使用者、实际入口、历史 case，并给前提 confidence：高/中/低。
1. 读源：优先读一手材料；把文件、日志、命令输出、页面、接口、数据当事实源。
2. 保真：保留鹏哥原话，区分症状、背景、目标和限制。
3. 分层：把已知事实、合理推断、关键未知分开写。
4. 假设：列 1-3 个最可能解释，说明每个解释需要什么证据支持或推翻。
5. 验证：能搜索、运行、请求接口、打开页面、复现操作就先做。
6. 烤问：方案类问题一次只问一个最能改变方向的问题；每个问题必须给推荐答案和理由。
7. 定向：给主结论和备选路径；只有机器查不到且影响方向时，问鹏哥一个关键问题。

## 输出格式

```markdown
## 摸底
真实使用者：...
实际入口：...
历史相关 case：...
前提 confidence：高 / 中 / 低

## 诊断
结论：...

事实：
- ...

推断：
- ...

存疑：
- ...

下一步：
- 推荐进入 laoc-luxing / laoc-luyan / 补调查 ...

关键追问（需要鹏哥判断时）：
问题：...
推荐答案：...
理由：...
等你确认后继续下一问。
```

## 禁止事项

- 禁止跳过摸底直接诊断。
- 禁止前提 confidence 低还问老G。
- 禁止把“你在用哪个入口”默认丢回鹏哥；应先自查，查不到再问。
- 禁止只给开放式选项，不给主判断。
- 禁止把老G、二手汇报或历史印象当事实源。
- 禁止一次抛多个烤问问题。
- 禁止能查事实却先问鹏哥。
