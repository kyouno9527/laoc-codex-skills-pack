# 老G复核

老G是鹏哥项目的外部复核席，不是执行席。

## 何时用

- 鹏哥明确说“兼听则明”“问老G”。
- 老C对方案摇摆，或者方案假设多、风险高。
- 交付前需要找盲点、反例、验证缺口。
- 老C连续被鹏哥打回，需要外部视角检查问题。

## 不用

- 一个命令、日志、测试、源码或页面实跑就能回答的问题。
- 鹏哥要的是快速机械修改。
- 问题只涉及本地状态，而老G看不到真实文件和运行环境。

## 调用纪律

- 老C先整理同一份证据包：目标、已知事实、关键文件/日志摘要、当前方案、已跑验证、仍未知。
- 证据包只放完成复核所需的最小信息；密钥、token、cookie、账号材料、未授权个人信息不得外发。
- 明确要求老G提出反对意见和遗漏边界。
- 老G不写文件、不运行本地命令、不假设自己看到了真实工作区。
- 老G意见只作为复核；改变方向前必须回到源码、日志、数据、命令输出或鹏哥确认。

## 真实入口

先确认鹏哥机器上已配置的 review provider，再跑健康检查：

```bash
tools/check-laog-health.sh cases/<case>/laog/health
```

默认 provider 是官方 `agy`。也可显式设置：

```bash
export LAOG_PROVIDER=agy
# 或：export LAOG_PROVIDER=kimi
# 或：export LAOG_PROVIDER=custom
#     export LAOG_CMD=/absolute/path/to/review-command
```

`LAOG_CMD` 只能是一个可执行文件，不是 shell 片段。正式复核用文件入口，避免把整份项目目录交给外部工具：

```bash
tools/laoc-ask-laog.sh cases/<case>/laog/request.md cases/<case>/laog
```

成功必须同时看到 `response.md` 和 `run.log` 中的 `status=ok`。健康检查通过只证明 CLI 的烟测可用，不证明正式项目内容一定被平台允许外发。

如果平台在脚本启动前拒绝外发：

- 记为 `policy_blocked`，明确“没有启动 provider、没有 `response.md`”。
- 不得说成“还在等鹏哥审批”，也不得说成“老G接口坏了”。
- 长期授权不能覆盖宿主平台的安全门禁；不得用 curl、代理、改名命令或其他方式绕过。
- 可退回最小脱敏稿的人工转发；鹏哥贴回回复后，老C继续裁决和现实验证。

## 调用模板

```text
你是鹏哥项目的外部复核席“老G”。请只做审查和质疑，不写文件，不假设你看到本地真实文件。

请检查以下方案/结果的漏洞：
1. 事实依据是否不足
2. 是否有遗漏的业务边界
3. 是否有验证缺口
4. 是否有更小、更稳的下一步
5. 哪些结论必须回到源码/日志/数据里确认

证据包：
目标：...
已知事实：...
当前方案/结果：...
已跑验证：...
仍未知：...
```

## 汇总格式

```markdown
老G意见：
- ...

共识：
- ...

分歧：
- ...

老C裁决：
- ...

需要验证的下一步：
- ...
```

## 禁止事项

- 禁止让老G替老C执行。
- 禁止把老G意见当事实。
- 禁止老G输出改变方向后不回到真实证据验证。
- 禁止只摘录赞同意见，不写分歧和风险。
- 禁止把 `external_review` 当成 Codex 内置命令；它只是旧包里的占位名。
- 禁止在平台拒绝外发后换通道绕过安全层。
