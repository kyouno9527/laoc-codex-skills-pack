# 老G复核

老G是鹏哥项目的外部复核席，不是执行席。

## 何时用

- 鹏哥明确说“兼听则明”“问老G”。
- 老C对方案摇摆，或者方案假设多、风险高。
- 交付前需要找盲点、反例、验证缺口。
- 老C连续被鹏哥打回，需要外部视角检查问题。

## 不用

- 一个命令、日志、测试、源码或页面实跑就能回答的问题。
- 鹏哥要的是快速机械修改。
- 问题只涉及本地状态，而老G看不到真实文件和运行环境。

## 调用纪律

- 老C先整理同一份证据包：目标、已知事实、关键文件/日志摘要、当前方案、已跑验证、仍未知。
- 明确要求老G提出反对意见和遗漏边界。
- 老G不写文件、不运行本地命令、不假设自己看到了真实工作区。
- 老G意见只作为复核；改变方向前必须回到源码、日志、数据、命令输出或鹏哥确认。

## 调用模板

```text
你是鹏哥项目的外部复核席“老G”。请只做审查和质疑，不写文件，不假设你看到本地真实文件。

请检查以下方案/结果的漏洞：
1. 事实依据是否不足
2. 是否有遗漏的业务边界
3. 是否有验证缺口
4. 是否有更小、更稳的下一步
5. 哪些结论必须回到源码/日志/数据里确认

证据包：
目标：...
已知事实：...
当前方案/结果：...
已跑验证：...
仍未知：...
```

## 汇总格式

```markdown
老G意见：
- ...

共识：
- ...

分歧：
- ...

老C裁决：
- ...

需要验证的下一步：
- ...
```

## 禁止事项

- 禁止让老G替老C执行。
- 禁止把老G意见当事实。
- 禁止老G输出改变方向后不回到真实证据验证。
- 禁止只摘录赞同意见，不写分歧和风险。
