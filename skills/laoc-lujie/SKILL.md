# laoc-lujie

laoc-lujie 负责把完成的工作变成可查、可续、可复用的状态。它只在验收成立后进行。

## 何时用

- laoc-luyan 裁决通过，工作需要收口。
- 鹏哥说“存档”“checkpoint”“沉淀”“记一下”“结案”“下次接着跑”。
- 任务会跨会话、跨平台、跨人员继续，需要明确下一步入口。
- 本次出现了可复用规则、决策、坑点或路径变化。

## 核心步骤

1. 确认裁决：引用 laoc-luyan 结论和关键证据；没有验收证据不结案。
2. 写状态：记录当前阶段、已完成、剩余、阻塞、下一步第一动作。
3. 写索引：长期会查的资料进入项目索引、共享记忆、case 索引或平台约定入口。
4. 写沉淀：只抽取可复用的 decision、lesson、pattern、surprise；流水账留在 case，不进常驻摘要。
5. 写迁移：如果入口、路径、流程、skill、架构或职责边界改变，同步更新对应状态文件。
6. 收束汇报：说明改了什么、验证了什么、沉淀到哪里、剩余风险是什么。

## 输出格式

```markdown
# Checkpoint

时间：YYYY-MM-DD HH:mm
状态：...
验收证据：...
已完成：...
剩余/风险：...
下一步第一动作：...
相关入口：...
```

可复用沉淀：

```markdown
Decision：...
Lesson：...
Pattern：...
Surprise：...
```

## 禁止事项

- 禁止不经验收直接结案。
- 禁止把流水账塞进常驻记忆。
- 禁止把敏感正文、密钥、cookie、私密授权材料写进共享沉淀。
- 禁止只写“完成了”，不写下次怎么接。
- 禁止用隐式入口、个人脑内记忆或不可追踪路径代替索引。
