鹏哥 / 老C 工作流 v2

这份包把最新 Shikagami 工作流蒸馏成鹏哥那边可直接安装的“老C”版本。核心不是多几个 prompt，而是一条事实驱动闭环：先查事实，再执行，再用现实证据验收，最后沉淀成下次可续跑的状态。

生成时间：2026-07-02 用户称呼：鹏哥 Codex：老C 主路由：知 -> 行 -> 验 -> 结

v2 新增重点

事实源分级
源码、日志、数据、命令输出、页面/API 实跑和鹏哥确认优先；外部模型只能做复核。
方案烤问
方案、设计、计划类问题一次只问一个最影响方向的问题，并给推荐答案和理由。
四层验收
不只看文件存在，还要看实质、接线和真实功能。
老G review-only
老G只审查和质疑，不写文件、不当事实、不替老C裁决。

老C主循环

阶段做什么关键纪律
知接诊、摸底、读源、分层、必要时烤问。能查事实先查事实，只问一个真正影响方向的问题。
行明确目标后小步执行，改文件、跑命令、做产物。先定边界和 DoD，不扩大范围，不覆盖别人改动。
验对照原需求做存在、实质、接线、功能四层验收。没有刚跑过或刚检查过的证据，不说完成。
结写状态、索引、可复用经验和下次续跑入口。只在验收通过后沉淀，不把流水账塞进长期记忆。

事实源顺序

  1. observed：源码、配置、日志、数据、命令输出、测试结果、页面/API 实跑、生成产物、鹏哥明确纠正。
  2. recorded：memory、cases、docs、skills、交付说明和历史 handoff。
  3. indexed：图谱、搜索索引、摘要；用于发现线索，不自动代表当前事实。
  4. external：网页、外部模型、第三方文档；只能做参考。
  5. backup：git、GitHub、归档快照；用于追溯和恢复，不默认是当前控制面。
如果来源冲突，老C先看最新可观察事实。

安装

git clone https://github.com/kyouno9527/laoc-codex-skills-pack.git
cd laoc-codex-skills-pack
bash scripts/install-laoc-workflow.sh /path/to/pengge-project

后续更新时在 laoc-codex-skills-pack 目录执行 git pull，再重新运行安装脚本。不想跑脚本时，可把 skills/、scripts/、mcp/ 复制进鹏哥项目，并把 AGENTS_LAOC_SNIPPET.md 追加到项目根 AGENTS.md。
